# Gitコマンドとは
このファイルはGitコマンドの理解を深めるために使うファイルである<br>
Gitコマンドとは以下の構成であり、英語のように正しく構文を理解する必要がある<br><br>
`コマンド`(gitなど)が<br>
`サブコマンド`(commitやpushなど)する<br>
`オプション`(-m,-v)詳しく<br>
`引数`目的語

## アップロード
ローカルリポジトリからリモートリポジトリにデータをアップする
### ステージング
挙げるファイルを選ぶこと<br>
```
git add ファイル名
```
`git add . `ですべてのファイルを選択
### コミット
変更の記録
```
git commit オプション ("コミットメッセージ") 
 ```
 `-v`オプションはコミットメッセージを打てる小ウィンドウが出る<br>
 `-m`オプションはコマンドライン上でコミットメッセージを打つ

 ### プッシュ
 リモートリポジトリへ反映、つまり同期
 ```
 git push origin ブランチ名
 ```

 ## ダウンロード
 ### プル
 リモートの進み具合をローカルに反映
 ```
 git pull
 ```
 

## ブランチ関連
ブランチとは作業スペースの分割である<br>
ホームページを作る作業、紹介ページを作る作業、バックエンドを作る作業、はたまたバグ修正作業などをそれぞれに干渉しないように、かつチームメンバーやそれぞれの作業内容を整理しやすいように分けることである
### 新しいブランチ作成
```
git branch ブランチ名
```
ブランチ作成しつつそのブランチに移るときは`checkout`サブコマンドに`-b`オプションを付ける
```
git checkout -b 新しいブランチ名
```
### ブランチの確認
現在あるブランチの確認
```
git branch
```

### ブランチの削除
```
git branch -d ブランチ名
```
-Dオプションに変えると強制削除
